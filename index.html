Lesson 1 - The web and HTML

What is the web?

The web is a giant network of computers that talk to each other using a protocol call HTTP. A webpage is document written in HTML (HyperTextMarkupLanguage). A web browser is a program that interprets and displays the info. The main pieces of the web are: users, the net and servers.

What is coding?

Coding is writing text in a language that a computer can understand. If you know how to talk to a computer, you can tell it do anything! That's pretty neat!!!

Computers ain't dat bright

Computers are very literal. They don't intepret the way a human does. This means one little typo can become a huge problem. This is an example of 'Technological Empathy'.

Inline vs block

Elements are either inline or block. Block elements form an invisible box around them. This is useful for diving your page up into sections for styling.

Examples of Inline elements: a, span, br, img and strong.

Examples of Block elements: p, form and div.

Thinking like a programmer

There are too many details to remember. Focus on the big ideas then know what and how to look up. Google to the rescue!!!

5 different types of thinking:

  1. Procedural - Creating clear and unambiguous instructions for a computer to follow
  2. Abstract - Finding similiarity, or generality, amongst seemingly different things (avoiding repetition)
  3. Systematic - Breaking a big problem down into smaller pieces (big-picture and decision-making) and how the pieces fit together
  4. Technological Empathy - Understanding how a computer works, what it is, and what it's good and bad at doing (understanding tools)
  5. Debugging - Systematic process of identifying the cause of a computer that doesn't work. If a program doesn't work, there is a mistake.

Here are the steps to fix the problem
1. Collect evidence (what makes the program fail?)
2. Generate theories (what caused this?)
3. Test theories (if theory is right, how do I find out?
4. Fix the problem

Lesson 3 - Adding CSS to HTML

Avoiding repetition

Avoiding mistakes and consistency:

When doing the same thing over and over it becomes easy to make a mistake. The benefit of CSS is that it allows you make changes to parts of the document with a few lines of code instead of going through and manually applying the style to each individual line. For example, with this page, by assinging each lesson and header a class, I'm able to set the style with just one bit of code. I didn't need to set it each time. Very, very useful.

What is CSS?

CSS stands for cascading style sheets. It is the code that applies the look to the HTML structure. It cascades throughout the whole document. Meaning it not only applies to element's that directly match, but element's children as well. This is inheritance. If there are multiple rules for a child, the most specific wins. You link to the CSS file in the head of the html doc. Put the CSS file and HTML file in the same folder.

Boxes, boxes, and boxes

Earlier I mentioned to see everything as boxes in HTML. This is not only helpful in developing structure but it makes it easier to style. There are 4 components to the boxes: Margin, border, padding and conent.

Because there are four components, sizing can be different. Also, browser display content differently. This will be important in the future.

Display: flex; allows divs to appear next to each other instead of stacked. Will use this to build a TOC (table of contents) that appears next to the conent. How do I make it stay static while content moves to make navigation easier?

Two techniques:

  1. use percents instead of pixels
  2. boxsizing: border-box

Code, test, refine

The strategy:

  1. Look for natural boxes
  2. Look for repeated styles and semantic elements
  3. Write HTML
  4. Apply style (biggest to smallest)
  5. Fix things

Most machines are designed to with a few functions in mind. To do more, you'll need to physically alter the machine.

Computers are different. They can be programmed to do anything we want as long as we can write a program that specifies a specific set of instructions. This is really, really cool imo. We are only limited by our creativity and then communicating that idea.

In English, people often make grammatical errors but we still can get at what they mean. Computers are literal Larry's, meaning they are not intuitive. If it isn't grammatically correct the code won't run.

Lesson 2 - Variables and Strings

What is a variable in Python?

Variables are a way to give names to values. For example: variable = 2.

This example is how you assing a value to a variable. You can change the value of the variabl later by re-assigning it a diffrerent value.

The = sign does not mean the same thing as it does math. Think of the equals sign as an arrow pointing to the left.

Variables are useful in a bunch of ways:

  • They improve code readability by using names that make sense
  • They allow us to store the value of important data
  • They give us the way to change the value of something

There is a difference between 2 + 2 and "2" + "2"... Anything with quotes around it is a string (just a sequence of characters). 2 + 2 would equal 4. "2" + "2" would be "22". This is what is called concatenation. It just means joining two strings together.

You can't add a string and an integer but you can multiply a string.

Lesson 3 - Functions

What is a function?

A function is something that takes input, does something to that input, and then produces an output.

What is the difference between making and using a function?

First you need to make a function by using a line of code starting with the keyword def then give the function a name followed by the function parameters in parentheses. The parameters are replaced by actual values when the function is used (called).

Then you specify in the "body" of the function the code that specifies what you want to do with the input parameters.

Next, you call the function by using the print statement followed by the name of the function followed by the values you want to input. Number of inputs needs to match then number of parameters. The value of the input is assigned of each parameter name in order and then the block is evaluated.

How do functions help avoid repetition?

Functions are tools that programmers can create and reuse forever. Once a function, always a function.

What happens when a function doesn't have a return statement?

The return keyword tells Python what the function should produce as an output. If there is not return keyword you get >>> None. No return.

Lesson 4 - If and While Loops

Using code to make decisions

In Python we use booleans (true or false) to decide whether or not to execute a block of code.

The 'if' function

The if function will only run if the expression is True. If the exression is False, the block is skipped.

Within the IF function an ELSE statement can be defined. So if the IF statement is false, then it will run the ELSE block of code. The ELSE is on the same indentation as the IF.

AND and OR

You can test to run a block of code if x AND y are True. Also, you can test if x OR y are True. However, if the first test is True it will not check the second test.

The MAX operator

You can find out the largest value in a set of inputs using max(a,b,c,d,etc..)

This is all we need to write programs!

With all the we've learned so far we could write any computer program. This means that all of programming breaks down to arithmetic and evaluating these statements.

Buuuut, most computer programming is done using pre-programmed procedures that will make it easier to write out bigger pieces of code.

Loops

A loop is a way to go over a pieces of code over and over.

Here we have the while loop. This continues to run WHILE the test is True. Watch out for infinite loops.

Break statements give you a way to stop a loop even if the test is True.

Lesson 5 - Structured Data

Lists

Lists are similar to strings, however strings a limited to text data. Lists can contain any kind of data. Lists can contain strings, numbers, lists or a combo of everything. Instead of using quotes to contain the data you use the square brackets dividing each element with a comma.

Mutation

Mutation means you're able to change the value of a list after it has been created. Strings are not mutable. This is an advantage of lists.

Aliasing

Aliasing is when you have 2 seperate names that refer to the same object. If you change one, you change the other.

Append

The append function will add ONE element to the end of a list. This means it mutates the list it is invoked upon. It does not create a new list.

Plus function

You can concantenate lists resulting in a new list of the concantenated lists.

Len Function

The LEN operator returns the length of a string or list. It returns the number of elements in a string and for lists it will return the number of elements.

For Loops

A for loop runs a block of code for each element in a list until all the elements have been evalutated.

Indexing Lists

If you search for a value it returns the first occurence in the list.

In function

This is a boolean operator? It will let you know if the value is in the list or not.

How to solve problems

The five steps for solving problems!

  1. Don't panic!
  2. What are the inputs?
  3. What are the outputs?
  4. Solve the problem!
  5. Simple mechanical solution (Don't optimize prematurely)
  6. Develop solution in incrementally small steps to test as you go
Remember that making progress is good!

Debbugging

The overview

Debugging is a scientific process of examining code to discover things you didn't expect.

It isn't just correcting errors or learning how mistakes were made.

It's unavoidable so its important to develop systems to solve problems.

1. Examine error messages when programs crash. The last line tells you what wrong. Read backwards to learn more about the problem. Google will be your friend.

2. Work from example code. If modified code doesn't work, commet it out and do step-by-step modifications.

3. Make sure examples work. Check to see if runs properly.

4. Check (print) immediate results. When code doesn't crash, but doesn't do what's intended, use print statements to find out why.

5. Keep and compare old versions. When you have a working version of code, save it before adding more code. This gives you something to come back to.
